# Product Strategy

## Current Read

The app is strongest when it feels like:

- a playable guitar game
- a guided skill path
- a song-feel practice tool

It is weakest when it feels like:

- abstract drills with no musical payoff
- fake versions of famous songs
- melody-only arrangements that have no weight

## Important User Feedback

For the songbook-style world:

- level 1 felt fine
- level 2 felt fine
- level 3 felt weak
- level 5 felt weak
- the Jingle Bells arrangement felt recognizable but still too light

Interpretation:

- recognition alone is not enough
- melody-only content often feels thin
- users want songs to have weight, pulse, and payoff

## Main Product Recommendation

Do not build for total beginners from zero.

Best target:

- late beginners
- returning players
- self-taught players who know a few basics

This user already knows:

- how to hold the guitar
- how to fret single notes
- maybe a few open chords or riffs

This user still needs:

- structure
- motivation
- real-time feedback
- songs and riffs that feel good quickly

Why this target is best:

- complete beginners often need more hand-holding than the app currently gives
- advanced players usually want deeper theory, transcription, improvisation, or real repertoire depth
- late beginners are the sweet spot for game-like progression and quick wins

## The Real Content Problem

The issue is not just "real songs vs original songs."

The issue is that some tabs feel too light.

Weight comes from:

- rhythmic identity
- bass-note anchors
- repeated root motion
- dyads or partial chords
- stronger phrase resolution
- anticipation and release

A melody by itself can be recognizable but still feel empty.

## Content Direction

The app should have 3 content lanes:

1. Guided Worlds
2. Songbook / Familiar Songs
3. Drill Library

### 1. Guided Worlds

This is the main curriculum.

Recommendation:

- 6 worlds total
- 8 core levels per world
- 2 boss levels per world

That gives:

- 10 levels per world
- 60 main levels total

Each world should have one job.

Suggested world plan:

1. Starter Riffs
2. Blues Phrasing
3. Anthem Leads
4. Heavy Riffs
5. Familiar Songs
6. Chords and Groove

### 2. Familiar Songs

This should not be called "Public Domain Songbook" in the product UI.

Better naming directions:

- Familiar Songs
- Playable Classics
- Campfire Songs
- First Standards
- Songs You Know

Recommended name:

`Familiar Songs`

Reason:

- easy to understand
- not legal-sounding
- broad enough for public-domain now and licensed songs later

### 3. Drill Library

The library should stay as the free-browse mode:

- riffs
- hooks
- songs
- warmups
- favorites
- recent

This supports retention better than worlds alone.

## Best Product Positioning

Position the app as:

`Learn guitar through playable riffs, familiar songs, and real-time feedback.`

Not:

- a full beginner course from zero
- a giant tab database
- a pure rhythm game

## Accessibility Strategy

To make the app more usable by more people:

- make every world start with 2 low-pressure levels
- show exactly why a level is worth learning
- keep song length short at first
- give one clear goal per level
- make success feel possible in under 3 minutes

Good level goals:

- "Hit 80% of the melody"
- "Keep the pulse steady"
- "Land the big jump cleanly"
- "Play the hook twice without breaks"

## Should We Add Chords?

Yes, but not immediately everywhere.

Recommendation:

- keep single-note play as the main loop for now
- add dyads and 2-note grips before full chord worlds
- then add simple open-chord rhythm levels later

Why:

- dyads add weight fast
- they sound fuller
- they are easier to detect than full strummed chord complexity

Best progression:

1. single-note melody and riffs
2. bass + melody accents
3. dyads
4. simple rhythm chords

## Why Some Song Levels Feel Weak

Song levels feel weak when they only trace the top melody.

To fix that, future song arrangements should use one of these templates:

### Template A: Melody + Root Anchors

- short melody phrase
- low-string root hit
- return to melody

### Template B: Hook + Dyad Punch

- melody start
- 2-note accent on strong beat
- continue melody

### Template C: Bass Walk + Melody Answer

- low note movement first
- melody replies after

These keep recognizability while making the guitar part feel fuller.

## Monetization Direction

Best first monetization:

- subscription SaaS

Do not start with ads.

Why:

- ads hurt practice flow
- music learning needs immersion
- subscriptions fit progression, sync, streaks, and content drops

Good paid value:

- cloud sync
- premium worlds
- familiar song packs
- weekly new content
- advanced feedback

## Current Best Path

1. Keep the current core worlds.
2. Rename the public-domain lane to `Familiar Songs`.
3. Rework the weaker song arrangements with more weight.
4. Expand to 10 levels per world over time, not all at once.
5. Add dyad-based levels before full chord content.
6. Build for late beginners and returning players.

## Immediate Next Build Recommendation

The next best concrete move is:

`Refine Familiar Songs so each arrangement has more weight.`

That means:

- rewrite the weaker levels
- add root-note support
- add dyad punches where possible
- keep recognizability high
- keep the arrangements easy enough for late beginners
